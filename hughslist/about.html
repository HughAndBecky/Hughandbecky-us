About Hugh's List

History

In the early 2000s, linguists and specialists working with language resources recognized the need for reliable ways to describe and share information about their collections. The Open Language Archives Community (OLAC) emerged to address this need by coordinating standards and aggregating metadata from participating archives.

For a brief period, a form hosted through the LINGUIST List allowed individuals to contribute records directly. When that option ended, record creation largely remained the responsibility of major archives, even though many researchers, community members, and other stakeholders continued to hold valuable knowledge about language resources that was not represented in institutional collections.

The need for accessible, high‑quality metadata has only increased as search engines and discovery tools rely more heavily on structured information. The OLAC Record Creator was developed to support this ongoing need by providing a user‑friendly way for individuals to prepare descriptive metadata records that can participate in broader discovery ecosystems.

Purpose

This application provides a way for individuals to prepare and submit descriptive metadata records for language-related resources. It is focused on OLAC compatable records, which are also more broadly compatible with the Dublin Core metadata standard. The application is designed to be user‑friendly and accessible, even for those without technical expertise in metadata standards.

Technology

The application performs processing in the browser and incorporates freely accessible data.

The code base is not open source, but the data produced is Open Data.

Contributors

Hugh's List was developed by Hugh Paterson III. For more information about Hugh's background and work, please visit his online profile.

Data Storage

The data submitted through this application to Hugh's List is Open Data. You can also use the forms and not submit the data to Hugh's List.

Resource metadata is persisted through Zenodo as a bulk download. It is made accessible via OAI-PMH.
