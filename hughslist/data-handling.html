Data Flow & Privacy

Data you enter

This application works with descriptive information that you type, paste, or import into on‑screen forms. When you choose to use external lookups or imports, the returned metadata is displayed in the browser and can be edited before you generate any output.

Data that stays in your browser

All metadata you enter or edit remains in your browser while you are using the application. The application does not create user accounts, maintain server‑side sessions, or store your records on a backend service (at this time). An ORCID login is required to access certain features, but this authentication process is handled entirely by ORCID and does not involve the application receiving or storing any of your credentials.

Data that never leaves your browser

The application does not send your draft records, file contents, or generated XML to any third party service.

When data does leave the browser

Data leaves your browser only when you initiate an action that requires contacting an external service. The common case is when an identifier is sent to a lookup service to retrieve metadata. In this case, only the identifier (e.g., an ISBN or OCLC number) is transmitted, and the returned metadata is displayed for your review and editing.

Only the minimal query information needed to complete the request is sent. Draft records are never transmitted. At a future time Hugh's List may offer optional features that allow you to save or share your records, but these features will be designed with open access in mind.

Data Processing

Processing of the information you enter takes place within your browser. No server‑side computation is required to build or validate your records. All transformations occur locally on your device.

Local storage and cookies

Hugh's List does not use cookies or local storage to save your data. Your records are stored only in memory while you are using the application.
